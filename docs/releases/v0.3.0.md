## Summary

v0.3.0 is the first feature-complete development milestone for Docker Topology Live. It delivers a read-only local Docker topology visualiser with live updates, opt-in metrics and diagnostics, host path privacy controls, and a fully offline browser UI (D3 vendored locally). The release is intended for local developer use only; it has not been validated for production or multi-user deployments.

---

## Highlights

- **Packaged Python project** — installable via `pip install -e .` with a `dtl` console script
- **Live topology graph** — D3 force-directed browser UI, updates via Server-Sent Events; polling fallback
- **Read-only Docker access** — no container stop/remove/prune, no volume or image operations
- **Opt-in metrics** — `--metrics` enables CPU, memory, network, and block I/O display with Metric Glow
- **Local diagnostics** — `--diagnostics` runs 14 deterministic local rules; no external API calls
- **Host path redaction** — `--redact-host-paths` suppresses bind mount source paths from all output
- **Offline browser UI** — D3 v7 vendored locally; no CDN request at runtime
- **Sample mode** — full UI demo without a Docker daemon; works on any machine with Python

## Installation

```bash
# Sample mode and basic installation
pip install -e .

# Live Docker scanning, events, and metrics
pip install -e ".[docker]"

# Run without installing
PYTHONPATH=src python app.py serve --sample
```

## Quick start

```bash
# Sample UI — no Docker required
python app.py serve --sample

# Live UI with metrics and diagnostics
python app.py serve --metrics --diagnostics

# Privacy-safe scan — bind mount paths redacted
python app.py scan --redact-host-paths

# Run diagnostics against sample topology
python app.py diagnose --sample
```

---

## Security posture

- Server binds to `127.0.0.1` by default
- CORS disabled by default; opt-in only via `--allow-cors`
- No Docker mutation APIs
- No external telemetry or AI API calls
- No outbound network connections except to the local Docker socket
- Secret-like label values redacted with `***REDACTED***`
- Bind mount source paths optionally redacted via `--redact-host-paths`
- D3 bundled locally; no CDN egress at runtime
- Browser UI avoids `innerHTML`; DOM manipulation uses `createElement`, `textContent`, and `classList`

---

## Known limitations

- Docker stats payloads vary across cgroups v1/v2 and Docker Desktop; some metric fields may be zero or absent
- Diagnostics rules are heuristic; false positives are possible in intentional configurations
- Browser UI tested manually; no automated browser tests
- No Prometheus export, no historical metrics, no Docker API-side event filters
- Production or multi-user deployments are out of scope; this is a local developer tool
- Windows and WSL2 environments have not been validated in CI
